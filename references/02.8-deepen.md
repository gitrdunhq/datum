# Phase 02.8: Deepen (Evidence Gathering)

You are in the Deepen phase. Your objective is to ground the abstract plan in reality by gathering codebase evidence and appending it to `TASKS.md` without changing the existing task structure.

## Instructions

1. Read `TASKS.md`.
2. For each complex task, gather codebase evidence using the best available tool:
   - **GitNexus (preferred):** Use `gitnexus_context("symbolName")` for symbol lookups and `gitnexus_query("concept")` for concept search. This returns process-grouped results ranked by relevance.
   - **OpenGrep (pattern matching):** Use for regex-based pattern searches across the codebase when GitNexus doesn't cover the query.
   - **grep (last resort):** Only use raw grep when both GitNexus and OpenGrep are unavailable.
   
   Search for:
   - Existing implementations of similar logic.
   - Project conventions (e.g., how dependency injection or routing is usually handled).
   - Known pitfalls from `docs/solutions/` if they exist.
3. Append a single section at the very end of `TASKS.md` titled exactly `## Research Findings`.
4. Inside that section, list the findings grouped by Task ID. Keep it concise.
   
   **Format Example:**
   ```markdown
   ## Research Findings
   
   ### TASK-001: Implement Auth
   - **Pattern**: See `auth/session.py:45` for existing middleware pattern.
   - **Pitfall**: JWT refresh races are known to occur; ensure atomic token swap.
   ```

5. **CRITICAL:** Do NOT modify the original tasks, delete anything, or change the hierarchy of the plan. You are strictly in append-only mode for `TASKS.md`.
6. Once the evidence is appended, run `datum doctor --phase deepen` to pass the gate.
7. Run `datum transition properties` to move to the next phase, and load the properties prompt.
