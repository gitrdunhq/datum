# Phase 02.3: Prior Art

**Goal:** Before anyone writes a line of code, search for existing solutions — open-source libraries, packages, internal repos, published techniques — that could replace, simplify, or inform each task in the plan. Reduce build-from-scratch to wrap-existing wherever possible.

## When it runs

After **Plan** (tasks exist), before **Triage** (which decides how deep to go). A strong prior-art hit can downgrade a task from "build" to "wrap a library," which changes triage routing and downstream scope.

## Inputs

- `TASKS.md` — the decomposed plan
- `docs/epics/<branch>/SPEC.md` — requirements context
- `tasks.json` — structured task list for machine parsing

## Instructions

### 1. Spawn parallel search agents

For each task (or group of related tasks), spawn a subagent to search **all four sources**. Fan out — these are independent lookups:

| Source | How to search | What you're looking for |
|---|---|---|
| **GitHub repos** | `gh search repos "<task keywords>"`, `gh search code "<function signature>"` | Existing implementations matching the task's intent. Prioritize repos with >100 stars, recent commits, compatible license (MIT/Apache/BSD). |
| **Package registries** | `pip index versions <pkg>` or web search `site:pypi.org "<concept>"` | Well-maintained packages that do what a task describes. Check: last release <12 months, >1k downloads/month, no critical CVEs. |
| **Internal repos (gitrdunhq)** | `gh search code "<keywords>" --owner gitrdunhq` | Code you've already written in turret, sgf-website, or other org repos. Reuse > reinvent. |
| **Web (docs, posts, papers)** | Web search for techniques, algorithms, design patterns | Approaches, not just code. "How to implement X" guides, reference architectures, known pitfalls. |

Each agent returns structured findings:

```json
{
  "task_id": "task-001",
  "findings": [
    {
      "source_type": "github|pypi|internal|web",
      "url": "https://...",
      "name": "library-name or repo-name",
      "relevance": "high|medium|low",
      "license": "MIT|Apache-2.0|...|unknown",
      "verdict": "use|wrap|reference|skip",
      "rationale": "One sentence: why this helps or why it doesn't fit.",
      "reduces_loc": 120
    }
  ],
  "recommendation": "build|wrap|vendor|skip",
  "recommendation_rationale": "..."
}
```

### 2. Verdict definitions

| Verdict | Meaning | Effect on task |
|---|---|---|
| **use** | Install the package and call its API directly. | Task becomes "integrate + test," not "build." Estimated LOC drops. |
| **wrap** | The solution is close but needs an adapter or subset. | Task scope narrows to the adapter/wrapper. |
| **vendor** | Copy specific files (with license attribution) into the repo. | Task becomes "adapt vendored code + test." |
| **reference** | The solution informs the approach but isn't directly usable. | Task unchanged but RED/GREEN agents get the reference as context. |
| **skip** | Nothing relevant found, or everything found is unsuitable. | Task unchanged. |

### 3. Quality gates on findings

Before accepting a finding:
- **License check**: GPL/AGPL findings get verdict `reference` at best, never `use` or `vendor` (license incompatibility). Flag and surface to user.
- **Maintenance check**: Last commit >2 years ago + <100 stars → `reference` at best, not `use`.
- **Security check**: Known CVEs or `npm audit`/`pip-audit` flags → surface, don't auto-accept.
- **Scope check**: A library that does 10x what the task needs → `reference` unless it's a well-known standard (e.g., requests for HTTP, pytest for testing).

### 4. Write outputs

**Standalone report** → `docs/epics/<branch>/PRIOR_ART.md`:

```markdown
# Prior Art Report

## Summary
- Tasks with actionable findings: N/M
- Estimated LOC reduction: ~X lines
- New dependencies proposed: [list with versions]

## Findings by Task

### task-001: [title]
| Source | Name | Verdict | Rationale |
|---|---|---|---|
| PyPI | some-lib 2.1 | use | Does exactly what AC1 requires, MIT, 5k stars |
| GitHub | user/repo | reference | Similar approach but different language |

**Recommendation:** wrap `some-lib` — reduces task from ~150 LOC to ~30 LOC adapter.

### task-002: [title]
No actionable findings. Build from scratch.
...
```

**Task annotations** → append to `TASKS.md` (append-only, like Deepen):

```markdown
## Prior Art

### task-001: [title]
- **Recommendation:** wrap `some-lib 2.1` (PyPI, MIT) — reduces ~150 LOC to ~30 LOC adapter
- **Reference:** https://github.com/user/repo — similar pattern, useful for RED note context

### task-002: [title]
- **Recommendation:** build (no suitable prior art found)
```

### 5. Dependency proposal protocol

If any task gets a `use` verdict (new dependency):
1. Check the dependency against `SEC-002` (published <90 days → reject)
2. Add it to a `## Proposed Dependencies` section in PRIOR_ART.md with: name, version, license, size, why it's needed
3. The **Triage gate** (next phase) surfaces proposed deps for human approval — no auto-install

### 6. Scope reduction

After all findings, if any task's recommendation is `use` or `wrap`:
- Update the task's `estimated_loc` in `tasks.json` (reduce it)
- Do NOT delete or merge tasks — that's the human's call at the Plan gate
- Log the delta: "task-001: 150 LOC → 30 LOC (wrapping some-lib)"

## Gate

Run `datum gate prior_art [--yolo]`

Validates:
1. `PRIOR_ART.md` exists in the epic directory
2. Every task in `tasks.json` has an entry (even if verdict is "skip — no findings")
3. `## Prior Art` section appended to `TASKS.md`
4. Any `use` verdict has a license field that is NOT GPL/AGPL (hard stop if found)
5. No proposed dependency published <90 days ago

On pass: state transitions to Triage.

Policy: `skippable_if_complete` — in yolo mode, if the gate passes, continue without human review.

## Outputs

| File | Location | Purpose |
|---|---|---|
| `PRIOR_ART.md` | `docs/epics/<branch>/` | Full findings report with verdicts and rationale |
| `TASKS.md` | repo root (appended) | Per-task prior-art annotations for downstream agents |
| `tasks.json` | repo root (updated LOC estimates) | Machine-readable scope reduction |
