# Phase 02.3: Prior Art

**Goal:** Before anyone writes a line of code, search for existing solutions — open-source libraries, packages, internal repos, published techniques — that could replace, simplify, or inform each task in the plan. Reduce build-from-scratch to wrap-existing wherever possible.

## When it runs

After **Plan** (tasks exist), before **Triage** (which decides how deep to go). A strong prior-art hit can downgrade a task from "build" to "wrap a library," which changes triage routing and downstream scope.

## Inputs

- `TASKS.md` — the decomposed plan
- `docs/epics/<branch>/SPEC.md` — requirements context
- `tasks.json` — structured task list for machine parsing

## Instructions

### 1. Spawn parallel search agents

For each task (or group of related tasks), spawn a subagent to search **all four sources**. Fan out — these are independent lookups:

| Source | How to search | What you're looking for |
|---|---|---|
| **GitHub repos** | `gh search repos "<task keywords>"`, `gh search code "<function signature>"` | Existing implementations matching the task's intent. Prioritize repos with >100 stars, recent commits, compatible license (MIT/Apache/BSD). |
| **Package registries** | `pip index versions <pkg>` or web search `site:pypi.org "<concept>"` | Well-maintained packages that do what a task describes. Check: last release <12 months, >1k downloads/month, no critical CVEs. |
| **Internal repos (gitrdunhq)** | `gh search code "<keywords>" --owner gitrdunhq` | Code you've already written in turret, sgf-website, or other org repos. Reuse > reinvent. |
| **Web (docs, posts, papers)** | Web search for techniques, algorithms, design patterns | Approaches, not just code. "How to implement X" guides, reference architectures, known pitfalls. |

Each agent returns **grammar-constrained structured output** via `datum.local_llm.structured()` using the Pydantic schemas in `datum/schemas.py`:

- **Search agents** → `PriorArtTaskResult` (contains a list of `PriorArtFinding`)
- **Security audit agents** → `SecurityAuditResult` (contains a dict of `SecurityCheckResult`)

These schemas are enforced at generation time via xgrammar (oMLX) or outlines (direct mlx_lm). The model literally cannot produce malformed JSON — the grammar constraint guarantees every field is present, every enum is valid, every string is within length bounds.

```python
from datum.local_llm import structured
from datum.schemas import PriorArtTaskResult

result = structured(prompt, schema=PriorArtTaskResult, model_id=model)
# result is a validated PriorArtTaskResult — no parsing, no retry on bad JSON
```

Schema shape (`PriorArtTaskResult`):
```json
{
  "task_id": "task-001",
  "findings": [
    {
      "source_type": "github|pypi|internal|web",
      "url": "https://...",
      "name": "library-name or repo-name",
      "relevance": "high|medium|low",
      "license": "MIT|Apache-2.0|...|unknown",
      "verdict": "use|wrap|vendor|reference|skip",
      "rationale": "One sentence: why this helps or why it doesn't fit.",
      "reduces_loc": 120
    }
  ],
  "recommendation": "build|wrap|vendor|skip",
  "recommendation_rationale": "..."
}
```

Schema shape (`SecurityAuditResult`):
```json
{
  "package": "some-lib",
  "version": "2.1.3",
  "verdict": "pass|accept_risk|reject",
  "checks": {
    "cves": {"status": "clear|flag|fail|n/a", "details": "..."},
    "supply_chain": {"status": "...", "details": "..."},
    "dep_tree": {"status": "...", "details": "..."},
    "license_tree": {"status": "...", "details": "..."},
    "code_review": {"status": "...", "details": "..."},
    "maintainer": {"status": "...", "details": "..."},
    "pinned": {"status": "...", "details": "..."}
  },
  "risk_summary": "Low risk. Well-maintained, MIT, no CVEs.",
  "conditions": []
}
```

### 2. Verdict definitions

| Verdict | Meaning | Effect on task |
|---|---|---|
| **use** | Install the package and call its API directly. | Task becomes "integrate + test," not "build." Estimated LOC drops. |
| **wrap** | The solution is close but needs an adapter or subset. | Task scope narrows to the adapter/wrapper. |
| **vendor** | Copy specific files (with license attribution) into the repo. | Task becomes "adapt vendored code + test." |
| **reference** | The solution informs the approach but isn't directly usable. | Task unchanged but RED/GREEN agents get the reference as context. |
| **skip** | Nothing relevant found, or everything found is unsuitable. | Task unchanged. |

### 3. Quality gates on findings

Before accepting a finding:
- **License check**: GPL/AGPL findings get verdict `reference` at best, never `use` or `vendor` (license incompatibility). Flag and surface to user.
- **Maintenance check**: Last commit >2 years ago + <100 stars → `reference` at best, not `use`.
- **Security check**: Known CVEs or `npm audit`/`pip-audit` flags → surface, don't auto-accept.
- **Scope check**: A library that does 10x what the task needs → `reference` unless it's a well-known standard (e.g., requests for HTTP, pytest for testing).

### 5. Security audit (mandatory when importing external code)

**Trigger:** ANY finding with verdict `use`, `wrap`, or `vendor`. If every finding is `reference` or `skip`, this step is skipped entirely — nothing enters the codebase, nothing to audit.

When triggered, spawn a **security audit subagent per dependency** (parallel, independent). Each agent runs the full checklist below and writes a structured verdict. The Prior Art gate **will not pass** until every audited dependency has a `pass` or `accept_risk` verdict.

#### 5.1 Audit checklist (per dependency)

| # | Check | How | Hard stop? |
|---|---|---|---|
| 1 | **Known CVEs** | `pip-audit <pkg>==<version>` or `gh api /advisories?ecosystem=pip&package=<pkg>` or web search `"<pkg>" CVE site:nvd.nist.gov` | Yes — any CRITICAL/HIGH CVE with no patched version = reject |
| 2 | **Supply chain signals** | Check: publish date (reject if <90 days per SEC-002), number of maintainers (single-maintainer = flag), download trend (sudden spike without release = flag), typosquat check (is this `requests` or `reqeusts`?) | <90 days = hard stop; others = flag for human |
| 3 | **Dependency tree depth** | `pip show <pkg>` → check `Requires:`; recursively count transitive deps. >20 transitive deps for a utility library = flag | No, but surface the count |
| 4 | **License scan (transitive)** | Check license of every transitive dependency, not just the top-level. One GPL deep in the tree poisons the whole chain. `pip-licenses --from=mixed --packages <pkg>` or manual check | Yes — GPL/AGPL anywhere in the tree = reject for `use`/`vendor` |
| 5 | **Code review (vendored only)** | For `vendor` verdict: read the actual source files being copied. Check for: `eval()`, `exec()`, `os.system()`, `subprocess.call(shell=True)`, obfuscated code, network calls to unexpected hosts, data exfiltration patterns, hardcoded credentials | Yes — any eval/exec/shell=True in vendored code = reject unless justified |
| 6 | **Maintainer reputation** | GitHub profile age, other maintained projects, org membership. Anonymous account + single package + recent creation = high risk | No, but flag for human |
| 7 | **Pinned version** | The proposed dependency must specify an exact version (`some-lib==2.1.3`), not a range. Lockfile must pin transitives. | Yes — unpinned = reject |

#### 5.2 Audit output (per dependency)

Each audit agent returns grammar-constrained output via `datum.local_llm.structured(prompt, schema=SecurityAuditResult)`. The schema enforces all fields, enum values, and string length bounds at generation time — no post-hoc parsing or retry needed. See `datum/schemas.py` for the full Pydantic model.

For `accept_risk` verdicts, the `conditions` array lists what the human must acknowledge:
```json
"conditions": ["Single maintainer — no bus-factor coverage", "12 transitive deps increases surface area"]
```

#### 5.3 Audit verdicts

| Verdict | Meaning | Gate behavior |
|---|---|---|
| **pass** | All checks clear. Safe to bring in. | Gate passes automatically (yolo) or with human review |
| **accept_risk** | Flags present but no hard stops. Risk is documented and bounded. | Gate **halts for human approval** even in yolo mode — `conditions` must be explicitly accepted |
| **reject** | Hard stop triggered (CVE, GPL in tree, eval in vendored code, <90 days old, unpinned). | Finding's verdict forcibly downgraded from `use`/`wrap`/`vendor` to `reference`. Task reverts to "build from scratch." |

#### 5.4 Where audit results go

- Appended to `PRIOR_ART.md` under a new `## Security Audit` section, one subsection per audited dependency
- Each audited dependency in the `## Proposed Dependencies` table gets an `Audit` column: `pass` / `accept_risk (N conditions)` / `reject (reason)`
- Rejected dependencies get a `⛔ REJECTED` prefix in the task annotations in TASKS.md, with a one-line reason

#### 5.5 Vendored code: extended review

For `vendor` verdict specifically (copying source files into the repo):
1. The audit agent reads EVERY file being vendored (not just the entry point)
2. Produces a `## Vendored Code Review` subsection in PRIOR_ART.md with:
   - File list with line counts
   - Any concerning patterns found (with file:line references)
   - Attribution block to include in the repo (original license text, source URL, version, date vendored)
3. The attribution block is a hard gate requirement — vendored code without license attribution = gate failure

### 6. Dependency proposal protocol

If any task gets a `use` or `wrap` verdict AND passes the security audit:
1. Add it to a `## Proposed Dependencies` section in PRIOR_ART.md with: name, version, license, audit verdict, transitive dep count, size, why it's needed
2. The Prior Art gate surfaces proposed deps: `pass` verdicts auto-clear in yolo, `accept_risk` verdicts always halt for human sign-off
3. No dependency is installed until the human approves — the gate enforces this

### 7. Scope reduction

After all findings, if any task's recommendation is `use` or `wrap`:
- Update the task's `estimated_loc` in `tasks.json` (reduce it)
- Do NOT delete or merge tasks — that's the human's call at the Plan gate
- Log the delta: "task-001: 150 LOC → 30 LOC (wrapping some-lib)"

## Gate

Run `datum gate prior_art [--yolo]`

Validates:
1. `PRIOR_ART.md` exists in the epic directory
2. Every task in `tasks.json` has an entry (even if verdict is "skip — no findings")
3. `## Prior Art` section appended to `TASKS.md`
4. Any `use` verdict has a license field that is NOT GPL/AGPL (hard stop if found)
5. No proposed dependency published <90 days ago

On pass: state transitions to Triage.

Policy: `skippable_if_complete` — in yolo mode, if the gate passes, continue without human review.

## Outputs

| File | Location | Purpose |
|---|---|---|
| `PRIOR_ART.md` | `docs/epics/<branch>/` | Full findings report with verdicts and rationale |
| `TASKS.md` | repo root (appended) | Per-task prior-art annotations for downstream agents |
| `tasks.json` | repo root (updated LOC estimates) | Machine-readable scope reduction |
