# Phase 02.5: Triage (Decision Matrix)

You have entered the Triage phase. Your objective is to dynamically route the pipeline by deciding if the `TASKS.md` plan warrants deep codebase research (`deepen`), or if it is trivial enough to skip straight to properties definition (`properties`).

## Instructions

1. **Delegate to Subagent:** You must spawn a subagent to perform this triage. This protects the primary orchestrator loop.
2. **Evaluate the Plan:** Instruct the subagent to read `TASKS.md` and evaluate it against the following rubric:
   - Does the plan modify security, authentication, or core data models?
   - Does it touch more than 3 files or span multiple domains?
   - Does it introduce a new dependency?
   - Does it require adhering to existing, complex architectural patterns?
   
   If **ANY** of these are true, the routing decision is `"deepen"`.
   If **ALL** of these are false (e.g., trivial CSS changes, typo fixes, simple UI string changes), the routing decision is `"properties"`.

3. **Write the Decision:** The subagent MUST write the decision to `.datum/routing.json` in the exact format:
   ```json
   {
     "decision": "<deepen OR properties>", 
     "reason": "<One sentence justification>"
   }
   ```

4. **Gate and Transition:** Once the subagent completes its task and `.datum/routing.json` exists:
   - Run `datum doctor --phase triage` to validate the gate.
   - Run `datum transition <decision>` (e.g., `datum transition deepen`) to branch the pipeline.
   - Load the prompt for the newly transitioned phase.
