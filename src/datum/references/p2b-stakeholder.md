# Pre-Dev Phase: Stakeholder Mapping (Optional)

**Goal:** Map stakeholder influence and align the proposed product against their needs.

## Context
This is an OPTIONAL phase. You are replacing the `wfc-stakeholder` skill. Only run this if the user requests it, or if the project has complex organizational boundaries, compliance needs, or enterprise deployment implications.

## Process
1. Read `docs/ideation/DISCOVERY.md` (and `COMPETITIVE.md` if present).
2. Map out primary, secondary, and tertiary stakeholders.
3. Identify their core needs, friction points, and potential blockers.
4. Produce an alignment strategy.

## Output
Create a `docs/ideation/STAKEHOLDERS.md` file summarizing the findings.
Transition to the `requirements` phase.
