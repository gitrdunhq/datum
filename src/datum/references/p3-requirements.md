# Pre-Dev Phase: Requirements

**Goal:** Transform the discovery findings into a structured Product Requirements Document (PRD).

## Context
This phase replaces `wfc-prd`. The output is the final "human-readable" specification of the product before it is distilled into an engineering ticket.

## Process
1. Read `docs/ideation/DISCOVERY.md`.
2. Draft user stories and acceptance criteria.
3. Define Non-Functional Requirements (NFRs).
4. Organize into a structured PRD.

## Output
Create a `docs/ideation/PRD.md` file containing the formal product requirements.
Transition to the `handoff` phase.
