# Playbook: AI-Optimized Git Workflows

**Goal:** Establish a robust version control foundation tailored for high-velocity, autonomous AI agent contributions in the DATUM factory. Traditional human workflows (like long-lived feature branches and squash-merging) break down at machine scale.

## 1. Local Sandboxing: Git Worktrees
To allow multiple agents or runs to execute the RED-GREEN-REFACTOR cycle concurrently on the same physical machine without filesystem lock contention or dirty state cross-contamination, DATUM strictly uses **Git Worktrees**.

- **Initialization:** Instead of `git branch`, the Orchestrator runs `git worktree add .datum/worktrees/<RUN_ID> -b <branch_name>`.
- **Execution:** All agent commands (`uv run test`, compiler checks) are executed inside `.datum/worktrees/<RUN_ID>`.
- **Garbage Collection:** Upon terminal failure or successful PR merge, the Orchestrator runs `git worktree remove --force .datum/worktrees/<RUN_ID>` and `git worktree prune`.

## 2. Trunk-Based Integration
DATUM operates on Trunk-Based Development. AI agents create extremely short-lived branches that are merged into `main` multiple times a day.

- **Pre-Review Rebase:** Agents MUST pull the latest `main` and rebase their worktree *before* generating the `diff.patch` for the Swarm Code Review. Reviewing stale code is a contract violation.
- **Merge Queues:** Because agents produce a high volume of PRs, a Sequential Merge Queue (e.g. GitHub Merge Queue) is strongly recommended. The queue acts as the serialization gatekeeper to prevent parallel merge conflicts.

## 3. Rebase and Merge (No Squashing!)
Agents are bound by strict **Conventional Commits** (e.g. `feat(scope): ...`, `test(scope): ...`).
Squash-merging a PR destroys this atomic commit history and renders `release-please` and automated changelogs inaccurate.
- **Rule:** All agent PRs MUST be integrated using **Rebase and Merge**. This flattens the history but preserves the individual atomic commits generated by the AI.

## 4. Production Safety: Feature Flags
To mitigate the risk of AI hallucination, regressions, or incomplete edge-case handling making it into production, DATUM relies on rapid decoupling of deployment from release.
- **Mandate:** All new agent-generated capabilities must be gated behind a Feature Flag (defaulting to OFF). 
- **Rollback:** If an AI ships a bug, the feature flag is toggled off in production. We do not use massive `git revert` commands unless the architecture itself is fundamentally corrupted.
