| CORR-004 | high | skills/src/datum-go.ts | 127 | Batch partitioning uses lanePlan.topological_order (line 125) to slice into fixed-size batches of MAX_BATCH=5 by sequential index. This ignores DAG wave boundaries: a task that depends on a task in the previous batch may be sliced into the same batch and will correctly detect the cross-batch dep via priorFailures—but completed cross-batch deps are never tracked in actCompleted before being needed by actFailures. Specifically, `priorFailures` accumulates failures across batches, but cross-batch *successes* are not checked, so a later-batch lane whose dependency completed in an earlier batch will see the dep absent from priorFailures and proceed, which is correct only if the batch loop is strictly sequential (which it is).
