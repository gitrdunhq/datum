| SEC-001 | medium | skills/src/datum-go.ts | 181 | epicBranch is parsed from an LLM sub-agent's JSON summary of `datum init --json` output and interpolated unsanitized into filesystem paths (`docs/epics/${epicBranch}/skeletons`, `docs/epics/${epicBranch}`) and into further natural-language prompts (lines 208-340) that direct sub-agents to construct and run shell commands. No validation that epicBranch matches an expected slug pattern occurs before use. | Validate epicBranch against a strict allowlist regex (e.g. `^[A-Za-z0-9_./-]+$` with no `..` segments) immediately after parsing, before using it in any path construction or downstream prompt; reject/throw on mismatch. |
| SEC-002 | low | skills/src/datum-closeout.ts | 78 | ctx.branch is interpolated directly into a grep -E regex pattern embedded in an agent-executed shell command (`grep -E '^[* ]+${ctx.branch}--'`) without escaping regex metacharacters, undermining the comment's claim of an 'exact prefix match, no other epics/runs'. | Escape regex metacharacters in ctx.branch before interpolation, or use `grep -F` with a literal prefix match instead of `-E`. |
| SEC-003 | low | scripts/test-count-gate | 45 | `--pattern-file` reads and inlines the full contents of an arbitrary caller-supplied file path (`PATTERN="$(cat "$2")"`) with no size/content validation or path restriction, and the resulting string is later fed to `grep -E`. | Bound the file size read, restrict the path to an expected temp-file location/prefix, and consider timing out or limiting grep to guard against a pathological regex causing ReDoS. |
| ARCH-001 | high | datum/cli.py | 310 | Abstraction leak: contextlib.redirect_stdout() silently suppresses output from downstream functions without their knowledge. The functions ensure_feature_branch(), _install_workflows(), and seed_state_docs.main() have no way to know they're running in quiet mode, violating Dependency Inversion Principle. | Add an explicit quiet_mode parameter to downstream functions instead of redirecting stdout. This makes the control flow explicit and allows functions to decide their own output behavior based on the calling context. |
| ARCH-002 | high | datum/cli.py | 315 | Tight coupling between error handling and output format: The _fail() nested function has conditional logic (json_output) baked into the error handling mechanism itself. This couples the error layer to the presentation layer, making it impossible to reuse error handling without including output formatting logic. | Separate error handling from presentation. Either: (1) Have _fail() raise a structured exception, then handle formatting in a separate layer, or (2) Use a strategy pattern where output behavior is injected as a dependency rather than embedded as a conditional. |
| ARCH-003 | medium | datum/cli.py | 339 | Scattered output-format conditionals (json_output appears in ~8 separate if-checks throughout init()). This creates tangled control flow where presentation concerns are interleaved with business logic, violating Single Responsibility Principle. | Extract output formatting into a separate abstraction (e.g., an OutputFormatter class or adapter) that encapsulates all JSON vs. text decisions. Pass this to helper functions instead of threading the boolean flag throughout. |
| ARCH-004 | medium | datum/skeleton_creator.py | 363 | Semantic coupling: The test_convention parameter controls not just output format but the fundamental semantics of the function (return type becomes None vs str vs str-path). The return type is polymorphic and implicit based on a string literal, making the function contract ambiguous to callers. | Use subtyping or overloads to make return type explicit: define separate functions (infer_test_path_flat, infer_test_path_directory, infer_test_path_docs_only) or use TypeScript overloads to encode the return type contract.
