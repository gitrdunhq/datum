# Review Report

## Findings

| ID | Severity | File | Line | Description | Suggestion |
|---|---|---|---|---|---|
| PERF-001 | high | datum/cli.py | 266 | subprocess.run() call to 'git status --porcelain=1' has no timeout. On large repositories or systems with I/O issues, this can cause datum init to hang indefinitely. | Add timeout parameter: subprocess.run([...], capture_output=True, text=True, timeout=10) |
| ARCH-001 | high | datum/skeleton_creator.py | 452 | infer_test_path() now returns str | None, but build_skeleton() calls it without handling None. When test_convention='docs-only', None is passed directly to template.format(path=None) at line 468, producing invalid traceability comments like 'Traceability: ... → None' | Either (1) handle None case in build_skeleton() by using a default value or skipping traceability, (2) add a test_convention parameter to build_skeleton() to propagate the convention through the call chain, or (3) never return None from infer_test_path() and raise an error instead |
| ARCH-002 | high | datum/cli.py | 309 | Tight coupling via contextlib.redirect_stdout(). The init() command assumes downstream functions (ensure_feature_branch, _install_workflows, seed_state_docs.main) write to stdout, then suppresses their output by hacking sys.stdout. This is fragile—if those functions change their output mechanism or add stderr output, this breaks silently. | Add explicit quiet/json output parameters to downstream functions (ensure_feature_branch, _install_workflows, seed_state_docs.main) instead of hijacking stdout at the caller. This inverts the dependency: each function controls its own output, not the caller. |
| ARCH-003 | medium | datum/cli.py | 315 | Nested _fail() function uses json_output from enclosing scope (closure coupling). Error type is also hardcoded as 'unsafe_branch_state' at line 317 regardless of actual error cause (used for no-commits, unsafe-state, and bootstrap-failed errors). | Extract _fail() to module level or a dedicated error formatter class. Add an error_type parameter to _fail() to distinguish between different failure modes, or use the decorator pattern to handle JSON/console formatting separately from error handling logic. |
| ARCH-004 | medium | datum/cli.py | 339 | Presentation logic scattered across function via multiple if-not-json_output guards (lines 339, 348, 357, 371, 389, 394, 397-400, 403-412). Violates Single Responsibility—business logic is entangled with output formatting. | Create a dedicated OutputFormatter class or use the Strategy pattern: business logic returns structured data (dicts/dataclasses), then a formatter transforms it to JSON or console output. This separates concerns and makes testing easier. |
| CORR-001 | high | datum/skeleton_creator.py | 452 | build_skeleton never threads the new test_convention parameter through to infer_test_path, so the directory/docs-only conventions added for #270 are unreachable from the actual skeleton-generation pipeline. build_skeleton calls infer_test_path(task_files, language, ac_id) with no test_convention argument, always defaulting to "flat". | Add a test_convention parameter to build_skeleton (sourced from task metadata) and pass it through to infer_test_path; add an integration test exercising build_skeleton itself with a directory/docs-only task. |
| CORR-002 | medium | skills/src/datum-go.ts | 163 | Act-phase bootstrap now runs the mutating `datum init --json` (can create/checkout a new branch, seed TICKET.md/workflows) instead of the old read-only git rev-parse prompt, with no check that HEAD is still on the epic branch established by earlier phases. If Act is resumed while on a protected branch, datum init will silently create a new unrelated branch and datum-go.ts will derive skeletonDir/epicDir from it, diverging from the real epic's artifacts instead of failing loudly. | Validate the epicBranch returned by datum init --json against .datum/pipeline-state.json's recorded branch when prior state exists, and throw a clear mismatch error instead of silently proceeding. |
