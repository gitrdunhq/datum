# TICKET: E10 — Reviewer (Phase C)

## Intent
Act as the final skepticism node before PR merge (ADR-0008). Evaluate the complete diff and passing tests generated by Phase B from an adversarial perspective. Assess regression risk and generate a final markdown review, blocking the lane if risk is deemed critical.

## Requirements
- `AdversarialReviewer`: Consumes the final lane outputs (the tested diff) and executes a (stubbed) model call to review the code.
- **Risk Assessment**: Determines if the change introduces unacceptable regression risk.
- **Output**: Produces a markdown formatted review and a final status of DONE or BLOCKED.

## Acceptance Criteria
- [ ] `AdversarialReviewer` analyzes a simulated diff and returns a structured review object.
- [ ] High-risk changes result in a BLOCKED status with a clear markdown explanation.
- [ ] Low-risk changes result in a DONE status.
- [ ] Strict TDD followed.
- [ ] `uv run pytest` green; tier-boundary guard passes.

## Constraints & NFRs
- `core` tier implementation (`src/datum_ax/core/reviewer`).
- Use stubbed model calls to mimic adversarial analysis. Pydantic schema for the review output.

## Classification
- Complexity: System · Scope: narrow · Ambiguity: low · Suggested route: feature
