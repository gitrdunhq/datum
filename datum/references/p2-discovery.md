# Pre-Dev Phase: Discovery

**Goal:** Expand the triaged problem statement into a comprehensive understanding of the domain, competitive landscape, and stakeholder needs.

## Context
This phase replaces the `wfc-ba` and `wfc-discovery` skills. 

## Process
1. Read `docs/ideation/TRIAGE.md`.
2. Map out stakeholders and perform lightweight competitive analysis.
3. Identify edge cases, assumptions, and risks that must be addressed before a formal PRD can be written.
4. **Sandboxing (wfc-playground):** If the feature includes a UI component, algorithm, or state transitions that are hard to visualize, generate a single-file, dependency-free HTML sandbox to test the concept visually. Save it to `docs/ideation/sandbox.html`.

## Output
Create a `docs/ideation/DISCOVERY.md` file summarizing the findings.
Transition to the `requirements` phase.
