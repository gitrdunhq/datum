# Pre-Dev Phase: Competitive Analysis (Optional)

**Goal:** Provide structured competitive analysis across market alternatives to ensure the product requirements address missing features or market gaps.

## Context
This is an OPTIONAL phase. You are replacing the `wfc-competitive` skill. Only run this if the user requests it, or if the domain clearly warrants competitive research (e.g., launching a new B2C app, a crowded SaaS space).

## Process
1. Read `docs/ideation/DISCOVERY.md`.
2. Map out at least 3 direct or indirect competitors.
3. Contrast their approaches, feature sets, and market positioning.
4. Distill how the proposed product can differentiate.

## Output
Create a `docs/ideation/COMPETITIVE.md` file summarizing the findings.
Transition to the `stakeholder` or `requirements` phase.
