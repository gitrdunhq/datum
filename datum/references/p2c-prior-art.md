# Pre-Dev Phase: Prior Art (Optional)

**Goal:** Search local memory and the internet to see how others have solved this problem. Do not reinvent the wheel—just make it rounder.

## Context
This is an OPTIONAL phase. Run this phase when tackling a problem that likely has established patterns, algorithms, or existing open-source libraries that can drastically reduce engineering effort.

## Process
1. Read `docs/ideation/DISCOVERY.md`.
2. Search local memory (e.g., past `solutions/`, `domain-wisdom.md`, or previous PRDs) for related work.
3. Search the internet for established patterns, APIs, libraries, or architectures that solve this exact problem.
4. Extract the best practices and identify what parts of the wheel we can borrow versus what we need to build custom.

## Output
Create a `docs/ideation/PRIOR_ART.md` file summarizing existing solutions, recommended libraries, and architectural patterns to leverage.
Transition to the next optional phase (`competitive`, `stakeholder`) or directly to `requirements`.
