# Pre-Dev Phase: Triage

**Goal:** Receive a raw user idea or business request and route it through initial complexity and priority checks before committing to discovery.

## Context
You are entering the Product Pipeline (Pre-Dev). The engineering lanes do not exist here. No code should be written.

## Process
1. Analyze the user's raw input.
2. Determine if the request is actionable.
3. Assess the priority.
4. Extract the core problem statement.

## Output
Create a `docs/ideation/TRIAGE.md` file capturing the raw request and initial assessment.
Transition to the `discovery` phase.
